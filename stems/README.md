# Stems — HaTchi-MaXchi

Package-level stems for the design system. They specialise framework stems
(especially **hypermedia-ssr**) for Hyperparts.

| Inherit from monorepo | `../../stems/` (dsl-first, agent-first, hypermedia-ssr, epistemic-layout, …) |
| Own | Hyperpart identity, composition, invention, layers, morph-safe hypermedia, details open intent, affordance disclosure chrome, selection-strip honest, shortcut-hint chrome, overlay light-dismiss, chrome vs protocol |

## Authority inside this package

1. **`stems/`** (this tree)
2. **`AGENTS.md`** — curriculum
3. **`docs/decisions/`** — ADR-style expressions of these stems
4. **`docs/agent/`** — playbooks (procedure)
5. **Machine maps** — `CONSUMER_MAP.md`, `CONTRACT_SURFACE.md`, dual-lock
6. Gallery / `MOCK_HTMX` — demo only

## Relation to `docs/decisions/`

Decisions are **expressions** (dated, statused). Stems are the **compressed claim**.
If they diverge, fix the expression; do not grow a second conflicting doctrine.
