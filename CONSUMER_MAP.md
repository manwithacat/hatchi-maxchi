# Hyperpart consumer map (reverse index)

Auto-generated by `tools/consumer_map.py`. Do not hand-edit.

Regenerate: `python packages/hatchi-maxchi/tools/consumer_map.py --write`

When you mutate a lower-level part (contract attrs, controller seams,
box metrics, submit shape), check **who consumes it** before shipping.
Hard embeds (`composes`) and blueprint uses are blast-radius. Soft
`composes_with` is agent guidance only. **Refused by** is deliberate
non-composition — do not wire that part in without flipping the registry
entry and completing the linked spike.

## How edges are declared

| Edge | Source | Meaning |
|------|--------|---------|
| `embedded_by` | `Hyperpart.composes` | Parent embeds this part in partial/exchange |
| `related_from` | `guidance.composes_with` | Soft agent link (not a runtime import) |
| `extends_parent` | `Hyperpart.extensions` | Extension controller rides this parent |
| `refused_by` | `Hyperpart.does_not_compose` | Parent resembles this part but uses a local primitive |
| `used_by_blueprint` | `Blueprint.composes` | Full-page motif includes this part |

## Explicit non-compositions (local primitives)

These are the high-risk lookalikes. CI locks (`require_substrings` /
`forbid_substrings`) keep the local implementation honest.

| Parent | Does not use | Seam | Reason | Spike |
|--------|--------------|------|--------|-------|
| `grid` | `combobox` | kind=select in-cell editor | bare <select class=dz-inline-edit-select> for density, morph survival (root._dzEdit + before/after-swap), and commit-on-change PUT — not the progressive-enhancement combobox overlay | `docs/spikes/combobox-in-grid-cell.md` |
